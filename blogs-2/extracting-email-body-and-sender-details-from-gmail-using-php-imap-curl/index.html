Extracting Email Body and Sender Details from Gmail Using PHP (IMAP & cURL)

Introduction

  • Automating email processing helps businesses handle customer inquiries, order confirmations, and contact form submissions efficiently.
  • Extracting relevant details from emails reduces manual efforts and improves response time.
  • This article demonstrates how to:
    • Connect to Gmail using PHP IMAP.
    • Retrieve unread emails and extract important details.
    • Process email content to fetch the sender’s name, email, subject, and message body.
    • Send extracted data to a CRM (FunnelMaker) via cURL API.

IMAP and Its Importance

  • IMAP (Internet Message Access Protocol) is used to fetch emails from a mail server.
  • Unlike POP3, IMAP does not delete emails from the server after retrieval.
  • It allows accessing emails from multiple devices simultaneously.

Setting Up Gmail for IMAP Access

  • Log in to Gmail.
  • Navigate to Settings > See all settings.
  • Select Forwarding and POP/IMAP.
  • Under IMAP access, choose Enable IMAP.
  • Click Save Changes.
  • Generate an App Password if using two-factor authentication (2FA).

PHP Code to Connect to Gmail via IMAP

<?php
$hostname = '{imap.gmail.com:993/imap/ssl/novalidate-cert}';
$username = 'your-email@gmail.com';
$password = 'your-app-password';

// Establish connection
$inbox = imap_open($hostname, $username, $password) or die('Cannot connect to Gmail: ' . imap_last_error());
  • $hostname specifies Gmailâ€™s IMAP server.
  • $username is the email address.
  • $password is the Gmail password or App Password if 2FA is enabled.
  • imap_open() establishes a connection to Gmailâ€™s IMAP server.

Fetching Unread Emails

$emails = imap_search($inbox, 'UNSEEN');

if ($emails) {
    foreach ($emails as $email_number) {
        $overview = imap_fetch_overview($inbox, $email_number, 0);
        $message = imap_fetchbody($inbox, $email_number, 1);
        echo "From: " . $overview[0]->from . "<br>";
        echo "Subject: " . $overview[0]->subject . "<br>";
        echo "Message: " . $message . "<br><hr>";
    }
}

imap_close($inbox);
?>
  • imap_search($inbox, 'UNSEEN') fetches unread emails.
  • imap_fetch_overview() retrieves email metadata (sender, subject).
  • imap_fetchbody() extracts the email message body.
  • imap_close() closes the connection to prevent resource leaks.

Extracting Specific Email Details

  • The email body contains raw text, requiring string processing to extract key details.
  • A function can be used to extract specific sections of an email:
function get_string_between($string, $start, $end) {
    $string = ' ' . $string;
    $ini = strpos($string, $start);
    if ($ini == 0) return '';
    $ini += strlen($start);
    $len = strpos($string, $end, $ini) - $ini;
    return substr($string, $ini, $len);
}

$sender_name = get_string_between($message, 'From:', 'Email:');
$sender_email = get_string_between($message, 'Email:', 'Subject:');
$email_subject = get_string_between($message, 'Subject:', 'Message Body:');
$email_body = get_string_between($message, 'Message Body:', 'This e-mail was sent from');
  • get_string_between() extracts content between specified keywords.
  • The function is used to get sender name, email, subject, and message body.

Sending Extracted Data to a CRM (FunnelMaker) Using cURL

  • After extracting email details, they can be sent to an external CRM for processing.
  • cURL is used to send an HTTP POST request with the extracted data.
function sendToCRM($name, $email, $subject, $messageBody) {
    $form_url = "https://app.funnelmaker.com/xml.pl";
    
    $data_to_post = array(
        'email' => "dev***jk1@funnelmaker.com",
        'password' => "dev*****1",
        'xml' => "<Contact><Name>$name</Name><Email>$email</Email><Subject>$subject</Subject><Message>$messageBody</Message></Contact>"
    );

    $curl = curl_init();
    curl_setopt($curl, CURLOPT_URL, $form_url);
    curl_setopt($curl, CURLOPT_POST, true);
    curl_setopt($curl, CURLOPT_POSTFIELDS, $data_to_post);
    curl_setopt($curl, CURLOPT_RETURNTRANSFER, true);

    $result = curl_exec($curl);
    curl_close($curl);

    return $result;
}

$response = sendToCRM($sender_name, $sender_email, $email_subject, $email_body);
echo "CRM Response: " . $response;

$form_url is the API endpoint of the CRM. curl_setopt() sets up the HTTP POST request. curl_exec() sends the request and retrieves the response.

Error Handling and Debugging

  • Check Connection Errors:
if (!$inbox) {
    die('IMAP Connection Error: ' . imap_last_error());
}

Verify API Response:

if (strpos($response, 'Success') !== false) {
    echo "Data successfully sent to CRM.";
} else {
    echo "Error: " . $response;
}
  • Enable cURL Debugging:
curl_setopt($curl, CURLOPT_VERBOSE, true);

Security Considerations

  • Use App Passwords instead of storing raw Gmail passwords.
  • Validate extracted data to prevent API errors.
  • Limit IMAP requests to avoid rate limits.

Performance Optimization

  • Batch processing: Process multiple emails in one script execution.
  • Caching: Store processed email IDs to prevent reprocessing.
  • Parallel execution: Use cron jobs to automate periodic execution.

Conclusion

  • PHP IMAP functions allow efficient email retrieval from Gmail.
  • Extracting sender details, subject, and message body is crucial for automation.
  • cURL API integration enables sending extracted data to a CRM system.
  • Implementing error handling and security ensures a robust solution.
